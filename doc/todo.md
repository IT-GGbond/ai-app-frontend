1. 用户初次打开页面/， 显示chatRecord，侧边栏显示历史聊天列表，页面初始挂载的时候根据路由id访问接口/detail获取聊天记录显示在页面上，如果没有id就是新的对话
2. 用户点击侧边栏的历史聊天记录，路由变为对应的/session/sessionId，和1同理，获取聊天记录显示在页面上 ✅
3. 用户点击chat组件的发送按钮，调用接口/llm，返回的数据是完整的聊天记录，包含历史聊天记录，同理2显示到页面上即可
4. 对3而言，如果是新对话初次调用发送接口，接口会返回聊天记录，此时要多做一步，就是重新获取侧边栏的历史聊天列表，这时接口会返回多一条数据，也就是当前新的聊天记录，并且路由变化为/session/sessionId

5. chatrecord更改

# 问题逐步改进
## sse请求
1. 原生 EventSource API缺点：仅支持get请求，url长度受限，无法自定义请求头
解决：使用npm包@microsoft/fetch-event-source
| 特性 | 原生 EventSource | @microsoft/fetch-event-source |
| :--- | :--- | :--- |
| HTTP 方法 | 仅 GET | 支持 GET, POST, PUT 等所有方法 |
| 请求头 | 不可自定义 | 完全支持自定义（如 `Authorization`） |
| 请求体 | 不支持 | 支持（如 JSON, FormData） |
| 错误重试 | 浏览器内置，不可控 | 完全由开发者控制，可自定义策略 |
| 连接控制 | 基础 | 支持 `AbortController` 主动中断 |

2. react的闭包陷阱，state快照
3. 流式传输滚动条问题
